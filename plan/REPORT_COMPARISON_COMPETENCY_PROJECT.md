## 3) Non-Negotiable Product Direction

The product is not a static roadmap viewer.
It must be a **Competency OS**:
- assess competency,
- execute learning,
- collect evidence,
- prove outcomes.

Core differentiators to protect:
1. Verified competency (self-claimed vs learned vs verified).
2. Role-based progression (Intern -> Fresher -> Junior -> Mid -> Senior).
3. Daily/weekly execution engine with labs and artifacts.
4. Outcome dashboard tied to delivery/operations metrics.

---

## 4) Admin Review Baseline (When Coder Team Finishes)

Use this checklist on every milestone report:

1. Data lineage
- Source -> ETL -> DB -> API -> UI is traceable.

2. No hardcode/no mock
- No runtime mock paths.
- No hardcoded business data in UI.

3. Backend-first truth
- All reads from DB-backed API/Server Actions.
- No fabricated fallback domain data.

4. UX quality
- Responsive, loading/error/empty states complete.

5. Quality evidence
- lint/typecheck/tests pass.
- Milestone risks and rollback notes documented.
